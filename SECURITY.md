# Security policy

## Supported version

Security fixes currently target the latest V2 Public release on the `main` branch.

## Reporting a vulnerability

Please do not publish exploitable security details in a public issue.

Use GitHub’s private vulnerability reporting feature from the repository’s **Security** tab. If private reporting is unavailable, contact the repository owner through GitHub and request a private channel without including exploit details in the initial message.

Include the affected version, reproduction steps, potential impact, and any suggested mitigation. Please allow a reasonable period for investigation and remediation before public disclosure.

## Scope

Relevant reports include script injection, unsafe file handling, credential exposure, microphone permission abuse, dependency vulnerabilities with a demonstrated impact, and ways that local-only user data could leave the device unexpectedly.
