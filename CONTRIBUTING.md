# Contributing to CTR Studio V2 Public

Thanks for helping make independent broadcasting more accessible.

## Before contributing

1. Search existing issues and pull requests.
2. Open an issue before starting a large feature or architectural change.
3. Keep V2 Public local-first, accessible, and usable without an account.
4. Keep paid-edition or private-service code outside this repository.

## Development

```bash
npm install
npm run dev
```

Open `http://localhost:3000`. Run the smoke test before submitting changes:

```bash
npm test
```

## Pull requests

- Keep each pull request focused on one change.
- Explain the user-facing outcome and testing performed.
- Do not commit music, credentials, personal data, generated builds, or local environment files.
- Preserve keyboard use, readable contrast, and responsive layouts.
- Test audio changes with headphones before using speakers or a live signal path.
